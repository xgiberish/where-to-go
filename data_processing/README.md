# data_processing

Data pipeline for building the travel-destination ML training dataset.

## Overview

Produces `ml/data/raw/destinations.csv` — 150 Asia-Pacific destinations with
justified travel-style labels and numeric features ready for ML training.

## Pipeline scripts

Run in order:

```bash
python data_processing/scripts/1_fetch_raw_data.py   # generate raw CSV from seed catalogue
python data_processing/scripts/2_clean_and_label.py  # clean, label, extract features
python data_processing/scripts/3_validate_dataset.py # validate schema, export ML CSV
```

### Script 1 — fetch raw data

Generates `data/raw/destinations_raw.csv` from a hardcoded 150-destination seed
catalogue (`DESTINATIONS` list in the script).  Each entry carries:
`destination_name, country, travel_style, budget_tier, climate, avg_temp,
best_season, tags`.

Synthetic reviews (50 per destination) are generated by cycling 15 sentence-frame
templates and seeding each frame with a destination tag keyword.  This is
**reproducible** — given the same tags and frames the output is deterministic.

### Script 2 — clean and label

1. Loads raw CSV, cleans whitespace and numeric fields (`clean_raw_row`).
2. Deduplicates on lowercased `destination_name`.
3. Runs `TravelStyleLabeler` to assign `travel_style` + `secondary_styles`.
4. Runs `FeatureExtractor` to compute `cost_index`, `safety_score`,
   `tourism_density`, `activities`.
5. Validates each row with `validate_labeled_row`; invalid rows are dropped and
   logged.
6. Writes `data/clean/destinations_labeled.csv` and `data/metadata/labeling_rules.json`.

### Script 3 — validate and export

1. Loads labeled CSV and validates every row against `LabeledDestinationRow`
   (Pydantic schema).
2. Exports passing rows to `ml/data/raw/destinations.csv` in `MLTrainingRow`
   format (`name,country,climate,travel_style,budget_tier,best_season,tags`).
3. Writes `data/metadata/validation_report.json`.

## Labeling methodology

### Algorithm

```
density(style) = total_keyword_occurrences_in_corpus / num_reviews

primary_label  = style with max density ≥ KEYWORD_THRESHOLD (0.15)
secondary      = other styles above threshold, sorted desc (up to 2)
fallback       = "budget" when no style clears the threshold
```

Normalising by `num_reviews` makes destinations comparable regardless of how
many reviews they have.  The threshold (0.15) means a keyword must appear at
least once every 6–7 reviews on average.

### Keyword sets

| Style       | Example keywords |
|-------------|-----------------|
| adventure   | trek, hike, climb, diving, rafting, volcano |
| relaxation  | beach, spa, massage, resort, onsen, yoga |
| culture     | temple, heritage, museum, shrine, palace |
| budget      | cheap, hostel, backpacking, street-food |
| luxury      | villa, fine-dining, 5-star, butler |
| family      | kids, child-friendly, zoo, aquarium |

Full keyword lists are in `config.py` (`ADVENTURE_KEYWORDS`, etc.) and mirrored
in `data/metadata/labeling_rules.json` after each pipeline run.

### Confidence levels

| Level  | Condition            |
|--------|----------------------|
| high   | ≥ 80 reviews         |
| medium | ≥ 50 reviews         |
| low    | < 50 reviews         |

### Justification

Keyword-density labeling was chosen over manual tagging because:
- **Reproducible**: same inputs always produce the same labels.
- **Auditable**: `labeling_rules.json` records per-destination densities.
- **Extensible**: adding new styles only requires adding keyword lists.
- **No API cost**: works offline with synthetic or real reviews.

## Feature extraction

| Feature          | Method |
|------------------|--------|
| `cost_index`     | Price-keyword cost estimates averaged → scaled 1–10 |
| `safety_score`   | safe-keyword / (safe + danger) mentions × 10 |
| `tourism_density`| Review-volume bucket ± crowding-keyword adjustment |
| `activities`     | Top-5 activity categories by keyword-hit count |

See `processors/feature_extractor.py` for full documentation and calibration
sources.

## Output files

| File | Description |
|------|-------------|
| `data/raw/destinations_raw.csv` | 150 destinations with synthetic reviews |
| `data/clean/destinations_labeled.csv` | Labeled + feature-extracted rows |
| `data/metadata/labeling_rules.json` | Keyword sets, threshold, per-destination densities |
| `data/metadata/validation_report.json` | Schema validation summary |
| `ml/data/raw/destinations.csv` | ML training input (name, country, climate, style, tier, season, tags) |

## Configuration

All tuneable parameters are in `data_processing/config.py` and can be overridden
with `DP_`-prefixed environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `DP_KEYWORD_THRESHOLD` | 0.15 | Minimum density to assign a label |
| `DP_MIN_REVIEWS_FOR_LABELING` | 30 | Skip destinations below this count |
| `DP_MAX_REVIEWS_PER_DESTINATION` | 100 | Cap for scraper (live mode) |
